INTENTS:
- leave_balance          → employee asks about their leave, vacation, sick days, or time-off balance
- leave_apply            → employee wants to apply for / take leave (annual, sick, personal). NOT asking about balance.
- policy_query           → employee asks a question answerable from HR policy documents (expenses, travel, conduct, benefits, etc.)
- software_provision     → employee requests access to a tool/system (Gitea, Mattermost, GitHub, Slack, etc.)
- access_request_status  → employee asks about the status of a previous access/provisioning request
- unsupported            → the question is outside the above categories

ENTITIES to extract (JSON, only include what is present):
- leave_type        → "annual", "sick", "personal", "parental", "long_service"
- leave_duration    → numeric value for leave application (e.g., 3, 0.5, 16)
- leave_unit        → "days" or "hours" (default: "days" if ambiguous)
- systems           → list of tools/systems mentioned
- team              → team or department mentioned for access
- topic             → the policy topic being asked about
- request_id        → specific access request ID (e.g., "AR-0001")

RESPONSE FORMAT — return valid JSON only, no explanation:
{{
  "intent": "<intent>",
  "entities": {{ ... }},
  "confidence": <0.0–1.0>
}}
